Introduction

What is JavaScript ?

  • JavaScript (often abbreviated as JS) is a High-level , interpreted programming language "runs line-by-line" that is most commonly used to make web pages interactive and dynamic .
  • In JavaScript, the interpreter is the JavaScript Engine inside the browser . How the interpreter works.
    • Reads your code
    • Compiles it just-in-time (JIT) into machine code
    • Executes it
  • Client-side scripting language: Runs directly in the user’s web browser without needing to be compiled.
  • Core web technology: Alongside HTML (structure) and CSS (style), JavaScript adds behavior to web pages.
  • Interactivity: Used for things like dropdown menus, form validation, animations, image sliders, and interactive maps.
  • Server-side: While originally designed for the browser, JavaScript can now run on servers (e.g., Node.js) and even in desktop and mobile apps.
  • JavaScript : can update and change both HTML and CSS.

Note : JavaScript is a hybrid language

High-level language

A high-level programming language is a language designed to be easy for humans to read, write, and understand, without worrying about hardware details.

Where we can type JavaScript ?

You can type and run JavaScript in many different places. Here are the most common places

1- console

  1. In Your Web Browser (The Easiest Way to Start) Browsers like Chrome, Firefox, and Edge have built-in JavaScript engines. You can run code directly in them.
  2. Browser Console (Best for Quick Tests) This is the fastest way to test a line or a small block of code.
  3. How to open: Chrome/Edge: Press F12 or Ctrl+Shift+J (Windows)

Comments

In JavaScript, comments are used to explain your code — they are ignored by the browser (not executed). There are two main types of comments

  • Single line comments start with //.
  • Multi-line comments start with /* and end with */.

"use strict" is a directive that tells JavaScript to run the code in “strict mode.”

Strict mode makes JavaScript more secure and less error-prone by enforcing stricter rules for writing code.

Variables

Variables = Data Containers

In JavaScript, variables are containers used to store data — like numbers, text, or objects in memory — that your program can use and change.

You can think of it as a box with a label — the label is the variable’s name, and the content inside is the value.

Example : let age = 47; age → variable that holds the number 47.

How can I name a variable (Rules and Conventions)

Rules (must follow)

  1. Variable names must start with:
    • A letter (a-z or A-Z)
    • A dollar sign ($) or underscore (_)
  2. After the first character, you can use letters, numbers, $ or _
  3. No spaces allowed.
  4. Case-sensitive → Name and name are different.
  5. Cannot use reserved words like let, var, function, etc.
  6. Conventions (good practices)
    • Use camelCase for normal variables → userName, totalAmount

In JavaScript, var , let , and const are all used to declare variables, but they behave differently .

What Is Scope?

Scope means the area in your code where a variable is accessible. In simple words: it defines where you can use a variable in your program. Not all variables can be used everywhere — their “lifetime” or “visibility” depends on where you declared them.

Types of Scope in JavaScript

  • Global Scope
  • Function Scope
  • Block Scope
Example
Type Description Declared With
Global Scope Accessible everywhere** in the program declared outside any function or block
Function Scope Accessible **only inside the function created by `var`, `let`, or `const` inside a function
Block Scope Accessible **only inside `{ }`** (like in `if`, `for`, etc.) created by `let` or `const`

JavaScript Output

JavaScript can "display" data in different ways:

  • Writing into an HTML element, using innerHTML or innerText .
  • Writing into the HTML output using document.write() .
  • Writing into an alert box, using window.alert() .
  • Writing into the browser console, using console.log() .
